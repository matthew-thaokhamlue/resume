Personal project · LLM workflow automation

PM work should be thoughtful, not manual.

A multi-LLM Python suite that automates release notes, Jira ticket drafting, API testing notes, and other recurring workflows, freeing time for actual product judgment.

Type Personal automation suite
Stack Python, APIs, LLMs
Providers OpenAI, Claude, Gemini
Outcome 12+ hrs/week saved

The problem

Too much PM time was trapped in formatting.

Release notes, Jira tickets, API testing notes, and recurring product operations all require consistency. But consistency should not consume 8-12 hours every week.

I built a multi-LLM Python toolset that uses OpenAI, Claude, Gemini, Jira APIs, and lightweight workflow apps to automate the repeatable parts while keeping the judgment layer human.

12hweekly manual PM work eliminated from recurring operations
90%reduction in manual formatting errors
15mrelease note creation time, down from roughly four hours
3LLM providers orchestrated across workflows

Workflow architecture

Agentic workflows for the boring work.

The suite parses structured and unstructured inputs, calls the right LLM for the job, and produces consistent artifacts: release notes, Jira ticket drafts, API test summaries, and PM operating documents.

The product instinct behind it is the same one I use at work: automate the parts where consistency matters, keep the human in the loop where judgment matters.

  • Saved 12+ hours per week across routine PM work.
  • Created 100% consistency in recurring documentation formats.
  • Reduced release note creation from four hours to fifteen minutes.
