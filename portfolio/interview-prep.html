Interview prep as an agentic workflow.

A multi-agent framework where agents parse your CV once, research the company, analyze the role, and generate stage-specific preparation documents with reusable context.

Type Open source framework
Agents Onboarding + interview prep
Pipeline 5 interview stages
Output Stage-specific prep docs

The problem

Interview prep is research, synthesis, and memory, all at once.

Good interview preparation requires more than reading a job description. You need your CV, the company’s product, business model, competitors, recent news, likely interviewer focus areas, and a stage-specific strategy.

I built a multi-agent Claude Code framework that turns that messy workflow into a reusable pipeline. Paste your CV once, name the company, then let agents research, structure, and generate prep materials for each stage.

5interview stages supported from HR screen to executive round
2core agents, onboarding and interview prep, with clean handoff files
0manual company research required after the pipeline starts
1xCV parsing step reused across every generated artifact

Agent system

A prep pipeline that compounds context.

An onboarding agent parses the CV into a canonical profile. An interview-prep agent then runs per stage, searches the web for company context, fetches job descriptions, researches interviewer backgrounds, and writes a structured prep document.

Every web-sourced point is tagged, uncertain findings are marked for verification, and each stage builds on prior context instead of starting from zero.

  • Generated stage-specific prep for HR, hiring manager, technical, team, and executive rounds.
  • Made company research autonomous while preserving source traceability.
  • Created reusable canonical files so the system remembers instead of re-parsing.
